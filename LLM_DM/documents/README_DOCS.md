# FIDD-Bench 文档索引

本文档汇总了 FIDD-Bench 项目的所有核心文档，方便团队成员快速查阅。

## 📚 文档列表

### 1. [需求文档 (Requirements)](./requirements.md)
*   **内容**: 项目目标、功能需求、非功能需求、目标用户、使用场景、业务流程。
*   **用途**: 明确“我们要不仅做什么”，作为项目开发的指导纲领。

### 2. [技术文档 (Technical Design)](./technical_design.md)
*   **内容**: 技术栈选型、系统架构图、模块划分、核心数据结构。
*   **用途**: 指导开发人员进行系统设计和代码实现。

### 3. [项目结构文档 (Project Structure)](./project_structure.md)
*   **内容**: 推荐的文件目录结构、各目录功能说明。
*   **用途**: 统一代码组织方式，确保项目结构清晰。

### 4. [开发规范文档 (Development Standards)](./development_standards.md)
*   **内容**: 界面交互规范、命名规范、注释规范、代码格式、Git 提交规范。
*   **用途**: 保证代码质量和团队协作的一致性。

### 5. [接口文档 (API Specs)](./api_specs.md)
*   **内容**: 内部 JSON 配置协议、CLI 命令行参数说明、认证方式。
*   **用途**: 定义模块间的通信标准和用户使用接口。
